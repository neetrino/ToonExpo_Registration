# Visual verification

Compare the rendered implementation with the available design source at the relevant viewport and state.

- **Structure:** content hierarchy, semantic regions, ordering, and reusable component boundaries.
- **Layout:** container width, grid/flex behavior, alignment, gaps, padding, and overflow.
- **Typography:** family, weight, size, line height, letter spacing, wrapping, and truncation.
- **Surfaces:** colors, borders, shadows, radii, imagery, and icons using approved tokens or assets.
- **Responsive behavior:** specified breakpoints plus narrow and wide states relevant to the task.
- **Interaction:** hover, focus, active, disabled, validation, keyboard navigation, and visible focus.
- **Data states:** loading, empty, error, long content, and realistic content density where applicable.
- **Runtime:** browser console, failed assets, hydration/layout shifts, and obvious accessibility issues.

Prioritize differences that affect hierarchy, usability, responsiveness, accessibility, or repeated visual rhythm. Document source ambiguities and tool limitations instead of inventing certainty.
